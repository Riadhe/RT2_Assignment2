# RT2 Assignment — Benchmarking 3D Mapping Pipelines for a Humanoid Robot

Research Track II (Prof. C. T. Recchiuto) — Riadh Bahri (S8335614), DIBRIS, University of Genova.

A statistical comparison of **NVBlox** (proposed) vs **OctoMap** (baseline)
for 3D mapping on the Unitree G1 EDU humanoid in NVIDIA Isaac Sim, with
RTAB-Map as a secondary reference. Repeated trials are simulated by a ROS 2
node, anchored to real measurements from the COGAR benchmark, then analysed
with paired t-tests in a Jupyter notebook.

## Source of the anchor measurements

The synthetic trials in this assignment are anchored to real single-run
measurements from a separate project, the **COGAR 3D-mapping benchmark**,
which ran the full NVBlox / OctoMap / RTAB-Map pipelines on the G1 EDU in
Isaac Sim. That project is the origin of the anchor values in
`data/anchors/anchors.yaml` (latency, coverage, density per framework and
scene) and is provided here only as supporting context — it is **not** the
deliverable for this assignment.

COGAR benchmark repository:
https://github.com/Riadhe/COGAR_3D_Mapping_with_G1_EDU_in_Isaac-Sim

## Repository structure

- `paper/` — the IEEE research paper (LaTeX sources + PDF)
- `notebook/mapping_analysis.ipynb` — runs the trials, statistics, and figures
- `ros2_ws/src/mapping_experiment/` — the ROS 2 node that simulates one trial
- `data/anchors/anchors.yaml` — real COGAR measurements + stated assumptions
- `data/results/` — per-trial CSVs generated by the notebook
- `figures/` — plots exported for the paper

## Requirements

- ROS 2 Humble
- Python 3 with `rclpy`, `pandas`, `scipy`, `matplotlib`, `pyyaml`
- Jupyter Notebook

## How to run

1. **Build the ROS 2 package:**
```bash
   cd ros2_ws
   colcon build --packages-select mapping_experiment
   source install/setup.bash
```

2. **Start the experiment node** (terminal 1):
```bash
   ros2 run mapping_experiment experiment_node
```

3. **Launch the notebook** (terminal 2) and run all cells top to bottom:
```bash
   cd notebook
   jupyter notebook mapping_analysis.ipynb
```
   The notebook sends trial requests to the node, collects the metrics into
   `data/results/`, runs the paired t-tests, and saves the figures used in the
   paper to `figures/`.

## Method summary

- **Design:** 2×2 factorial (framework × scene), N = 20 trials per condition.
- **Proposed vs baseline:** NVBlox (GPU TSDF/ESDF) vs OctoMap (occupancy octree);
  RTAB-Map kept as a secondary reference.
- **Metrics:** map-update latency (ms) and reconstructed coverage (m³).
- **Statistics:** paired t-tests (shared per-trial scenario seed makes the design paired).
- **Hypotheses (stated before data):**
  - H1 — NVBlox has lower latency than OctoMap in the large scene.
  - H2 — the two are comparable in the small scene.
  - H3 — OctoMap achieves greater coverage in the large scene (LiDAR range).

## Reproducibility

All random seeds are fixed in `metrics_model.py`, so re-running the notebook
reproduces identical results. Data are synthetic but anchored to the real
COGAR measurements; the distributional assumptions are stated in
`data/anchors/anchors.yaml` and in Section IV of the paper.
